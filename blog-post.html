Introducing InterceptX: The Ultimate Modern Alternative to ModHeader for HTTP Modifications

As web developers, API engineers, and security auditors, we spend a significant portion of our time inspecting and tweaking HTTP traffic. For years, extensions like ModHeader have been the go-to utility for modifying request and response headers on the fly.

However, as the browser extension landscape transitions fully to Manifest V3—bringing stricter security, better performance, and tighter permission rules—many developers are looking for a modern, lightweight, and local-first alternative.

Enter InterceptX.

What is InterceptX?

InterceptX is a high-performance, compact Chrome extension designed to give you complete control over browser network requests. Built from the ground up on Manifest V3 using the declarative declarativeNetRequest API, it is fast, secure, and preserves your battery life by running lightweight matching rules inside the browser engine itself.

Whether you need to bypass CORS policies, simulate mobile user-agents, override security headers, or redirect API endpoints to your local development server, InterceptX does it all with a premium, glassmorphic UI.

Key Features at a Glance

If you are familiar with ModHeader, you will feel right at home with InterceptX—but with several modern upgrades:

1. Request & Response Headers

Inject, append, or strip headers on outgoing requests or incoming responses:
  • Set: Override an existing header or add a new one (e.g., custom auth tokens or Origin).
  • Append: Add values to existing headers (like Accept or Cookie).
  • Remove: Strip headers to test omitted key behavior.

2. URL Redirections

Need to redirect API endpoints or override file routes? InterceptX features a built-in regex redirect engine (using RE2 syntax).

Redirect matching patterns using capture groups (e.g. from https://api.production.com/(.*) to http://localhost:3000/\1).

3. Granular URL & Domain Filters

Don't leak modified headers to all websites. InterceptX offers three types of scoping filters:
Match: Applies rules only if the request URL matches a wildcard pattern (e.g. *://api.dev.local/*).
Exclude: Bypasses rule execution on static directories/assets (e.g. */static/*).
Domain: Restricts modifications to requests initiated from a specific domain hostname only (maps to initiator domain).

4. Resource Type Filters

Scope modifications to apply only to specific request formats. Filter rules for XHR/Fetch, Document/main_frame, Stylesheet, Script, Image, and more.

5. Multi-Profile Management

Create isolated rulesets for different development environments (e.g. "Dev", "Staging", "CORS Bypass"). Switch, clone, rename, or delete profiles instantly.

6. Privacy First & Local-Only

Unlike services that track telemetry or collect logs, InterceptX is 100% local. Your header configurations and rules are saved locally on your browser using chrome.storage.local. No analytics, no trackers.
Did you know?

By leveraging the native web browser declarativeNetRequest API, InterceptX registers routing and header modification directives directly with Google Chrome's engine. This yields near-zero performance overhead, unlike Manifest V2 extensions which require intercepting and pausing active JavaScript threads for every single network call.

Getting Started in 3 Simple Steps

Getting up and running with InterceptX takes less than a minute:

1

Install the Extension

Add InterceptX directly from the Chrome Web Store.

2

Add your First Rule

Open the extension popup, click + Request Header, and enter a key like X-Custom-Header with your desired value.

3

Activate & Intercept

Toggle the master Enable switch in the top-right header to Active. The status indicator will turn green and your modifications are immediately applied.

Need to manage complex setups? Just click the Dashboard button (top-right of the popup) to launch the dual-column workspace in a full browser tab.
